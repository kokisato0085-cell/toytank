# ToyTank

スマホのブラウザ（PWA）で遊べる **2D見下ろし戦車バトル**。
弾は壁で跳ね返り（跳弾）、地雷を仕掛け、敵戦車を全滅させる。
ソロ、またはオンライン2人協力（デュオ）で同じステージに挑める。

> 「はじめてのWii / Wii Play」収録の「戦車！」へのオマージュ作品です。
> 名称・画像・音はすべて独自に用意します（既存IPの素材は使用しません）。
> 画像素材（タイル・戦車など）は **Google Gemini で生成**したものを使用しています。

## 特徴
- **タッチ操作（PC切替対応）**：左スティックで移動、右エイムパッドを引いて離した瞬間に発射。PCではマウスカーソル照準にも切替可能。
- **跳弾の物理**：弾は壁で反射。敵AIは遮蔽物の向こうのプレイヤーを、外壁・内壁を使った反射（バンクショット）で狙ってくる。
- **色分けされた敵タイプ**：色ごとに速度・思考（待機/接近/カイト/追跡）・弾速・反射回数・地雷の有無などが異なる10種以上。BFS経路探索で壁を避けて動く。
- **地雷・破壊可能な壁**：地雷は時限で爆発し戦車も壁も巻き込む。もろい木の壁は爆発で壊れる。
- **データ駆動ステージ ＋ 自作エディタ「ToyTank Maker」**：マップと敵配置を JSON で持ち、エディタでマウス作成できる。複数ステージを順番に遊ぶキャンペーンに対応。
- **演出と効果音**：戦車スプライト・タイル画像、撃破演出、キャタピラ跡、効果音（発射/跳弾/爆発/走行音ほか）。
- **フレンドリーファイアあり**：味方の弾も味方に当たる（協力プレイ向けの設計）。

## 技術スタック
- クライアント：TypeScript + HTML5 Canvas + Vite（PWA）
- リアルタイムサーバー（段階2のオンライン協力以降）：Go
- テスト：Vitest（クライアント）、`go test`（サーバー）

## 開発ノート（技術的障壁 ↔ 解決策）
実装を進める中で直面した課題と解決を、ここに随時追記していく。

- **敵のバンクショットが当たらない** ↔ 場外4面の鏡像法では内壁を絡めた反射を表現できなかった。実弾と同じ反射物理を軌道シミュレーションでなぞり、全方位を粗探索→精探索して命中する角度を選ぶ方式に変更。弾の反射回数（例：2回反射）にも対応。
- **自弾どうしが消し合う／自爆する** ↔ 同一斉射の弾を同じ「グループ」で識別し、同グループは相殺しないが別グループ（自弾含む）は相殺するルールに。地雷は起動猶予を過ぎると設置者も巻き込む形に整理。
- **タイル画像の継ぎ目・四隅の黒抜け** ↔ 床はベース色で塗ったうえで1枚を全面に敷いて継ぎ目を消し、壁・穴は不透明部分を自動で切り出してマスにフィット。
- **戦車スプライトを敵タイプ色で塗り分け** ↔ 白〜グレーのシルエットに色を乗算（陰影を保持）。背景除去で残る緑フチはピクセル判定で透明化。
- **効果音の鳴り出しが遅い／自動再生制約** ↔ 読み込み時に先頭無音を検出して常にスキップ。最初のユーザー操作で再生を解除。走行音はフェードでON/OFFするループ再生に。
