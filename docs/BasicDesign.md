# ToyTank 基本設計書

[POLICY.md](POLICY.md) の大方針を具体仕様に落とす。数値は調整可（小方針）。
段階1（ソロ・オフライン）と ToyTank Maker（エディタ）を対象とする。段階2（オンライン）は概要のみ。

---

## 1. ゲーム概要
見下ろし2Dの戦車バトル。プレイヤー戦車を操作し、跳弾と地雷を使って敵戦車を全滅させる。
1発で破壊される緊張感（攻撃側・被弾側とも）。ソロ、または段階2でオンライン2人協力。

---

## 2. 座標・マップモデル
- ワールド座標は**ピクセル（px）**。原点は左上、x は右、y は下方向。
- マップは**グリッド（マス目）**。`cols × rows` 個のセル、1セル `cell` px（既定 64）。ワールドサイズ = `cols*cell × rows*cell`。
- **壁はマス目単位**。**戦車・弾・地雷はマス目に縛られず連続座標で自由に動く**（移動の滑らかさを保つ）。
- セル `(col,row)` の中心ワールド座標 = `((col+0.5)*cell, (row+0.5)*cell)`。
- タイル種別：`0=床` / `1=壊せない壁（鋼）` / `2=壊せる壁（ブロック）` / `3=穴`。
  - **穴**：戦車は通れない（壁と同じ扱い）が、**弾・射線・爆風は通過する**。落下はしない。1グリッド単位。

### 更新モデル
- 実時間ゲームだが、**固定タイムステップ（既定 60Hz）でシミュレーションを更新**し、描画は補間する設計とする。
  - 理由：段階2でサーバー権威の固定tick同期へ拡張しやすくするため（決定論寄りに保つ）。

## 3. ステージデータ形式（JSON）
ゲーム本体と ToyTank Maker が共有する唯一のステージ表現。

```jsonc
{
  "name": "stage-01",
  "grid": { "cols": 20, "rows": 15, "cell": 64 },
  "tiles": [                      // rows 行 × cols 列。0=床 1=鋼 2=ブロック 3=穴
    [1,1,1, /* ... cols 個 ... */ 1],
    /* ... rows 個 ... */
  ],
  "players": [                    // 開始位置（セル座標）。[0]=P1必須, [1]=P2(デュオ用)任意
    { "col": 2,  "row": 12 },
    { "col": 3,  "row": 12 }
  ],
  "enemies": [                    // 敵配置（セル座標）
    { "col": 16, "row": 3,  "pattern": "stationary" },
    { "col": 10, "row": 5,  "pattern": "mover" }
  ]
}
```

ゲーム本体は、ToyTank Maker が localStorage に保存したステージ（`toytank.stage.<name>`）または同梱のサンプルを読み込んで遊ぶ。URL `?stage=<name>` で対象を指定する。

### 制約（バリデーション）
- `tiles` は `rows × cols` の矩形。値は 0/1/2 のいずれか。
- マップ外周は鋼壁（1）を推奨（弾・戦車の場外防止）。エディタは外周を自動で鋼にするオプションを持つ。
- `players[0]`（P1）は必須。`players[1]`（P2）は任意。
- 敵は1体以上。
- プレイヤー・敵の開始セルは**床（0）**であること。
- 戦車の開始セルどうしが重ならないこと。

## 4. ToyTank Maker（ステージエディタ）
クライアントアプリ内のページ（例 `/editor`）。Canvas にグリッドを描き、マウス／タッチで編集する。

### 機能
- **グリッド設定**：`cols` `rows` `cell` を変更（既定 20×15×64）。
- **ツール**（パレットで切替）：
  - 壁ペイント：床 / 鋼 / ブロック / 穴（クリック＆ドラッグで連続塗り）
  - プレイヤー配置：P1 / P2（各1つ。再配置で移動）
  - 敵配置：止まる(stationary) / 動く(mover)（複数可）
  - 消しゴム（床に戻す・配置物を削除）
- **外周を鋼で囲む**ワンクリックボタン。
- **入出力**（OSのファイル操作に依存せず往復できるよう複数手段を用意する）：
  - **JSONテキスト欄**：現在の状態の §3 JSON を常時表示（コピー可）。欄に貼り付けて「テキストを反映」で読み込み。
  - **ファイル書き出し／読み込み**：`.json` のダウンロード／ファイル選択での読み込み。
  - **ブラウザ内保存（localStorage）**：名前を付けて保存し、保存名の一覧から読み込み・削除。キーは `toytank.stage.<name>`。
- **検証表示**：§3制約を満たさない場合は警告（書き出しは可能、ゲーム読込時に弾く）。

### 段階1での位置づけ
ToyTank Maker を**先に実装**し、これで作った「標準ステージ1枚」をゲーム本体の動作確認に使う。

## 5. 操作（大5の具体化）
横持ち・全画面。画面下に2つの操作系を描画する。
- **左スティック（移動）**：押した方向へ戦車本体が移動。アナログ（傾きで速度）。
  - **戦車らしい旋回**：車体の向き(heading)を入力方向へ `tankTurnRate`(約90°/0.5s) で旋回させてから進む。前進と**真逆（バック）はキャタピラなので旋回不要＝即移動**。横方向への新規発進は旋回ぶん遅れる（停止→別方向は約0.5s）。移動中の方向転換は止まらず曲がる。
- **右エイムパッド（照準＆射撃）**：
  - ドラッグ中：パッドの向きへ砲塔を向け、**直進の弾道予測線**を表示（§9）。
  - **離した瞬間に1発発射**（その向きへ）。
- **即撃ちタップ**：右パッドを引かずに画面を軽くタップ → **進行方向**（停止時は最後に向いた方向）へ1発発射。
- **地雷ボタン**：押すと現在位置に地雷を設置（§7）。
- 砲塔の向き：エイム中はパッド方向。非エイム時は進行方向に追従。

## 6. 弾・命中・跳弾・フレンドリーファイア
- 弾は**直進**（速度 `bulletSpeed` 既定 380px/s、サイズ基準 `bulletRadius` 既定9px）。各戦車の**画面内同時発射数に上限**（既定5）。見た目は進行方向を向いた先端のとがった細長い形状。
- **跳弾**：弾は壁／場外境界で反射（軸平行反射＝当たった面に応じて vx か vy を反転）。**反射回数上限** `maxBounces`（既定1）。上限を超える反射で消滅。
- **壊せるブロック（2）・鋼（1）・場外境界に命中**：いずれも**反射**（上限まで）。**弾では壊せるブロックを破壊しない**（破壊は地雷の爆発のみ）。
- **穴（3）の上は通過**（反射しない）。射線・爆風も穴を通る（壁＝鋼/ブロックのみが弾・射線・爆風を遮る）。
- **戦車に命中**：その戦車を**破壊**し、弾は消滅。
- **フレンドリーファイア**：弾は敵味方・自機を問わず戦車に当たる。
  - 自爆防止の猶予：発射後 `selfGrace`（既定150ms）は**発射者自身には当たらない**。猶予経過後は、跳ね返った自分の弾でも当たる。
- **弾同士の相殺**：弾と弾が接触したら（敵味方問わず）**両方消滅し、小さな爆発エフェクト**を出す。判定は両半径の和（`2*bulletRadius`）以内。

---

## 7. 地雷
- 戦車は地雷を設置（**同時設置数上限** `maxMines` 既定2）。半径 `mineRadius` 既定18px。
- **起動猶予** `armDelay`（既定0.5s）経過後にアクティブ化（中心が赤く点灯）。
- **起爆条件**：(a) 信管 `fuse`（既定6s）経過、(b) **設置後に弾が当たる**と即起爆、(c) 他の爆発に巻き込まれて**誘爆**。
  - ※「戦車の近接で起爆」は不採用。
- **警告表示**：起爆 `mineWarn`（既定2s）前から、光（中心）とオーラ（外周リング）が点滅する。
- **爆発**：半径 `blastRadius`（既定 2.0*cell）内の対象に作用。**戦車を破壊**（FF：敵味方を問わず当たるが、**その地雷の設置者だけは無傷**＝自爆しない）、**壊せるブロックを破壊**、**他の地雷を誘爆**。
  - **壁は越えない**：爆心と対象の間に壁（鋼・壊せる壁）があれば、その対象には効果が及ばない（対象自身が壊せる壁の場合、その壁自体は遮蔽に数えない）。
  - **多重貫通しない**：1回の爆発の影響は**破壊前のタイル**で確定する（壊す壁・巻き込む戦車を先に判定→最後に破壊）。重ねた壊せる壁は手前1枚だけが壊れ、奥は守られる。

---

## 8. 衝突・移動
- **戦車 vs 壁/穴**：戦車を半径 `tankRadius`（既定24px）の円とし、床以外（鋼1・ブロック2・**穴3**）の矩形と円で判定。めり込まないよう**軸別スライド**。穴も戦車は通れない（落下はしない）。
- **戦車 vs 戦車**：自機は敵を半径 tankRadius の円の障害物として扱い貫通を防ぐ。**移動型の敵も「壁＋自機＋他の戦車」を避けて動く**（互いに重ならない・軸別スライド）。
- **戦車 vs 場外**：場外セルもソリッド扱いで境界外へ出さない。
- 1セル幅の通路（64px）を直径48pxの戦車が通れる。

---

## 9. プレイヤー弾道予測線
- 右エイムパッドのドラッグ中のみ表示。
- 戦車から照準方向への**直進レイ**を、**最初の壁／場外／戦車に当たる位置まで**描く。
- **反射は描かない**（跳弾は自分の腕で読む）。

## 10. 敵AI（大4の具体化）
- 対象：最も近いプレイヤー。
- **射線判定（LOS）**：敵→対象の線分が壁セル（1/2）と交差するか。
  - **クリア**：対象へ直接照準して発射（MVPはリード無し）。
  - **遮蔽あり**：**バンクショット**を試みる。
- **バンクショット（反射1回）**：対象を壁面で鏡像化し、敵→鏡像 が当該壁まで遮られず、かつ反射後 壁→対象 も遮られなければ、鏡像方向へ発射（＝1回反射で対象に当たる射線）。
  - MVPは**場外境界4面での1回反射**を対象とする（内部の鋼壁での反射は将来拡張）。
- 発射弾の `maxBounces` は1（実際に跳ねる）。
- 行動パターン：
  - **stationary**：移動しない。**直射＋バンクショット**。発射間隔 `enemyCooldownStationary`（既定2.6s・遅め）。
  - **mover**：移動する。**直射のみ（バンクショットなし）**、**照準にばらつき** `enemyAimJitter`（既定±0.12rad）。発射間隔 `enemyCooldownMover`（既定1.8s）。
    - **行動軸**を一定間隔（`behaviorMin`〜`behaviorMax`）でランダム切替：**戦闘**（接近）／**退避**（離れる・自機が `enemyNear*1.5` 内のときだけ）／**無目的**（徘徊）。重みは概ね 戦闘0.6/徘徊0.3/退避0.1。
    - **壁を擦らず回り込む**：希望方向が塞がれていたら角度を段階的にずらして通れる向きへ進む（軸スライドはしない）。他の戦車・自機とも重ならない。
    - **砲塔は常に自機を向く**（移動方向と分離。壁際でも向きがブレない）。
  - **近接時**：自機が `enemyNear`（既定280px）内にいると発射間隔を `nearFireMult`（×0.55）に短縮＝攻撃が激しくなる（両タイプ共通）。
### 敵タイプ（データ駆動・`stage/enemyTypes.ts` のレジストリ）
- 敵は **タイプ（色）ごとの設定値** を持つ：色／速度(0=静止)／行動性格(guard/approach/kite/balanced/chaser)／発射間隔／発射数／**salvo(同時扇状) or 逐次連射**／反射回数／弾速／バンク可否／**サイズ倍率scale**／照準ばらつき／地雷数／**HP**／透明。
- ステージの `enemy.pattern` はタイプキーを参照。リザルト・区切り画面の「色×数」、撃破集計もタイプ別に自動対応。
- **連射**：`salvo=false`(既定)は同じ発射口から逐次速射（`burstGap` 最小0.13s）。`salvo=true`は複数門の同時扇状発射（ボス）。
- **HP**：被弾/爆風で1減り0で破壊（途中は小ヒット表示）。被弾リスポーンで生存敵のHPは全回復。
- **サイズ**：`scale` で当たり判定・描画を拡大（ボス=2倍）。発射は**機体半径の外**から（大型機の自滅防止）。
- 実装済タイプ：stationary/mover（旧）＋ wood/gray/darkgreen/yellowgreen/yellow(地雷4)/pink(3連射)/purple(5連射バンク地雷2)/black(超高速追跡2連射ミサイル地雷)/boss(2倍・HP5・5門同時)。残り white(透明) は次。
- 移動AIの補助ルール：
  - **全行動（戦闘/退避/徘徊）でBFS経路探索**：目的セル（戦闘=自機/退避=離れた床/徘徊=ランダム床）への「次の一歩」へ進み、**壁を正確に避ける**（0.3s毎に再計算）。
  - **壁越しに突っ込まない**：守備型(balanced/kite)は戦闘中に射線が壁で切れ、かつ自機が `enemyClose`(150px)より遠ければ退避へ切替（攻撃型は退かず回り込む）。
  - **ハマり脱出**：同じマスに2秒以上留まったら無目的（目的地を選び直す）へ。
  - **砲塔の向き**：射線が通っている時だけ自機へ向け、普段は進行方向を向く。
  - **自分の地雷を警戒**：地雷を置くタイプは、自分の地雷の爆風圏へは踏み込まない（離れる方向は可）。設置者は自分の地雷では無傷。
- 弾の相殺は**異なる陣営の弾（自弾↔敵弾）どうしのみ**（同陣営の連射は相殺しない）。
- ※バンクショットの射角計算（鏡像法）は実装の難所。まず場外4面の1回反射に限定して堅実に作る。

## 11. 残機・進行・勝敗（大4の具体化）
- 全戦車（プレイヤー・敵）とも**1発／爆発で破壊**。
- **クリア**：敵を全滅。
- **ソロ**：残機3。**被破壊で残機-1 → 区切りポーズ（`respawnPause` 既定1.2s「ミス！残機×N」）後、自機だけスタート地点に復活**。
  - **倒した敵・壊した壁・進行状況はそのまま維持**（被弾では巻き戻さない）。設置中の地雷・飛行中の弾はクリア。
  - **生き残った敵は初期位置(home)へ戻す**（向き・発射クールダウンもリセット）。倒した敵は復活しない。
  - **残機0でゲームオーバー**。倒した敵などが完全に元に戻るのは「ゲームオーバー／手動リスタート／次ステージ」のときだけ。
  - **ゲームオーバー時リザルト**：色別（敵パターン別）の戦車アイコン×撃破数を表示。撃破数はランを通して集計し、リスタートでリセット。
- **キャンペーン進行**：ステージの敵を全滅でクリア → 次ステージへ（**残機は引き継ぐ**）。最後をクリアで全クリア。リスタート（R/ボタン）は最初のステージから・残機リセット。
  - 各ステージ開始時に**区切り画面**（`introPause` 既定2.2s）：「ステージN」・出現する敵（色×数）・残機を表示してから開始。
- **デュオ（段階2）**：片方破壊→**復活なしで待機**（相方生存中は続行）。両方破壊（全滅）で**ステージやり直し**。

## 14. 演出（キャタピラ跡・撃破印・大破演出）
- **キャタピラ跡**：戦車（自機・移動型の敵）が `trackGap`(9px) 進むごとに、薄い2本のトレッド線を床に残す（最大 `maxTracks`、超過で古いものから消去）。**被弾（残機減）でリセット**＝継続しない。完全リセット（ゲームオーバー/手動リスタート/次ステージ）でも消える。
- **撃破バッテン印×**：戦車がやられた座標に×印を残す。**敵は白固定**（視認性のため暗いふちどり）。完全リセットまで残る（被弾リスポーンでは消えない）。
  - 段階2：プレイヤー/味方は**その戦車の色**の×にする。
- **大破演出**：戦車が破壊されると、その位置に大きな爆発＋周囲に小爆発（`deathFx` 0.85s）。
  - 自機被弾時は **dying 状態**で演出を見せてから「ミス！区切り→復活」または「ゲームオーバー（リザルト）」へ。演出中は中央表示なし・敵/弾は一時停止。
  - 敵撃破時はプレイ継続のまま同じ爆発を出す。
