# ToyTank 基本設計書（中方針・小方針）

[POLICY.md](POLICY.md) の **大方針** を具体仕様へ落とす層。本書は2階層で構成する。

- **中方針（中N）**：各システムの設計判断（「こう作る」）。各見出しに親 **`（← 大X）`** を併記する。
- **小方針（小N-n）**：調整可能な末端の数値・閾値・挙動パラメータ。各 中 の末尾に列挙し、**値の正は §13 マスター表（およびコード）** を参照する（値はここで二重管理しない）。

段階1（ソロ・オフライン）と ToyTank Maker（エディタ）を対象とする。段階2（オンライン）は概要のみ。
節番号は 大方針との対応を保つため固定し、`§N` 相互参照はそのまま 中N を指す。

## 1. ゲーム概要（← 大1・大4）
見下ろし2Dの戦車バトル。プレイヤー戦車を操作し、跳弾と地雷を使って敵戦車を全滅させる。
1発で破壊される緊張感（攻撃側・被弾側とも）。ソロ、または段階2でオンライン2人協力。

### 1.1 画面構成（アプリシェル：タイトル／モード選択／設定）（← 大9）
ページを開くと**タイトル（モード選択）画面**を表示し、選択後にゲーム/エディタ等へ遷移する。
画面状態（`title` / `settings` / `customlist` / `game` / `editor`）を持つ薄いシェル層で管理する。

- **中1.1-a タイトルのメニュー項目**
  - **ソロ**：同梱の公式20面キャンペーンを遊ぶ。
  - **チュートリアル**：基本操作の練習（§15）。**ソロの直下に小さく**配置。
  - **Co-op マルチ**：段階2。今は **Coming Soon**（押せない／「準備中」表示）。
  - **PvP**：最終目標。今は **Coming Soon**。
  - **自作ステージで遊ぶ**：localStorage の自作ステージを一覧から選んで遊ぶ。
  - **ToyTank Maker**：自作ステージを作る（エディタ `editor.html`）。
  - **設定**：全体音量・ミュート・**フルスクリーン切替**・**操作ガイド**（将来：操作方法のカスタマイズ枠）。
- **中1.1-b 公式ステージと自作ステージの分離（重要）**
  - **公式キャンペーン**＝コード（`campaignStages.ts`）が唯一の出所。**読み取り専用**で、エディタや localStorage からは**改変できない**。
  - **自作ステージ**＝localStorage（`toytank.stage.<name>`）。エディタで作成・編集し、「自作ステージで遊ぶ」で再生。
  - 旧仕様（localStorage のキャンペーンが公式を上書き／`?seedcampaign` で公式を localStorage へ書き出し）は**廃止**。公式と自作は混ざらない。
- **中1.1-c ゲーム開始後の画面（端末で出し分け）**
  - **スマホ**：CSSでビューポート全体を覆う**没入表示**にし（実全画面APIに非依存＝iOSでも有効。Androidは加えて Fullscreen API＋横ロックも試行）、キャンバスを画面いっぱいにフィット。**黒帯部分もタッチ有効＝左右どこでもパッドを置ける**。**画質は devicePixelRatio で高精細化**。**縦持ちは「横にして」案内を全面表示**（タイトル含む＝ホーム画面も横画面に統一）。没入時の操作（音量／リスタート／タイトル／全画面解除）は**右上の⚙ギアに集約**、💣は右端中央。
  - **PC**：**ダークテーマ**・**キャンバス拡大・中央寄せ**。操作ガイドとフルスクリーン切替は**設定に集約**（操作セレクタは廃止＝端末で自動判定）。全画面時は左上バーを隠し**右上⚙ギアに集約**。
- **中1.1-d タイトル背景デモ**：戦車AI同士のバトルを自動再生（アトラクトモード・無音・全滅で次面へループ）。
- **中1.1-e リザルト選択**：全クリア／残機ゼロで「**もう一度プレイ／タイトルへ**」を選べる。
- **中1.1-f 全体テーマ**：ダークテーマで統一。
- **中1.1-g 設定の永続化**：音量・ミュート・操作モードは localStorage に保存。

## 2. 座標・マップモデル（← 大1・大8）
- **中2-a 座標系**：ワールド座標は**ピクセル（px）**。原点は左上、x は右、y は下方向。
- **中2-b グリッド**：マップは**グリッド（マス目）**。`cols × rows` 個のセル、1セル `cell` px（小→§13、既定64）。ワールドサイズ = `cols*cell × rows*cell`。
- **中2-c 連続座標で動く対象**：**壁はマス目単位**。**戦車・弾・地雷はマス目に縛られず連続座標で自由に動く**（移動の滑らかさを保つ）。
- セル `(col,row)` の中心ワールド座標 = `((col+0.5)*cell, (row+0.5)*cell)`。
- **中2-d タイル種別**：`0=床` / `1=壊せない壁（鋼）` / `2=壊せる壁（ブロック）` / `3=穴`。
  - **穴**：戦車は通れない（壁と同じ扱い）が、**弾・射線・爆風は通過する**。落下はしない。1グリッド単位。

### 中2-e 更新モデル
- 実時間ゲームだが、**固定タイムステップ（既定 60Hz）でシミュレーションを更新**し、描画は補間する設計とする。
  - 理由：段階2でサーバー権威の固定tick同期へ拡張しやすくするため（決定論寄りに保つ）。

**小方針（← 中2、調整可・正は §13／コード）**：小2-1 `cell` セルサイズ・小2-2 シミュレーション周波数（既定60Hz）。

## 3. ステージデータ形式（JSON）（← 大8）
ゲーム本体と ToyTank Maker が共有する唯一のステージ表現。

```jsonc
{
  "name": "stage-01",
  "grid": { "cols": 20, "rows": 15, "cell": 64 },
  "tiles": [                      // rows 行 × cols 列。0=床 1=鋼 2=ブロック 3=穴
    [1,1,1, /* ... cols 個 ... */ 1],
    /* ... rows 個 ... */
  ],
  "players": [                    // 開始位置（セル座標）。[0]=P1必須, [1]=P2(デュオ用)任意
    { "col": 2,  "row": 12 },
    { "col": 3,  "row": 12 }
  ],
  "enemies": [                    // 敵配置（セル座標）
    { "col": 16, "row": 3,  "pattern": "stationary" },
    { "col": 10, "row": 5,  "pattern": "mover" }
  ]
}
```

- **中3-a 出所の分離**：公式キャンペーンは `campaignStages.ts`（コード）が唯一の出所＝**読み取り専用**。自作ステージは ToyTank Maker が localStorage（`toytank.stage.<name>`）へ保存し「自作ステージで遊ぶ」で再生（§1.1 の分離方針）。旧 `?stage` / `?seedcampaign` は廃止。
- **中3-b 既定グリッド**：**同梱の公式20面は横長16:9＝28×15**（エディタの既定は 20×15）。

### 中3-c 制約（バリデーション）
- `tiles` は `rows × cols` の矩形。値は 0/1/2/3 のいずれか（0=床/1=鋼/2=ブロック/3=穴）。
- マップ外周は鋼壁（1）を推奨（弾・戦車の場外防止）。エディタは外周を自動で鋼にするオプションを持つ。
- `players[0]`（P1）は必須。`players[1]`（P2）は任意。
- 敵は1体以上。
- プレイヤー・敵の開始セルは**床（0）**であること。
- 戦車の開始セルどうしが重ならないこと。

**小方針（← 中3、調整可）**：小3-1 エディタ既定グリッド（20×15×64）・小3-2 公式面の既定グリッド（28×15）。

## 4. ToyTank Maker（ステージエディタ）（← 大8・大7）
クライアントアプリ内のページ（例 `/editor`）。Canvas にグリッドを描き、マウス／タッチで編集する。

### 中4-a 機能
- **グリッド設定**：`cols` `rows` `cell` を変更（既定 20×15×64）。
- **ツール**（パレットで切替）：
  - 壁ペイント：床 / 鋼 / ブロック / 穴（クリック＆ドラッグで連続塗り）
  - プレイヤー配置：P1 / P2（各1つ。再配置で移動）
  - 敵配置：止まる(stationary) / 動く(mover)（複数可）
  - 消しゴム（床に戻す・配置物を削除）
- **外周を鋼で囲む**ワンクリックボタン。
- **入出力**（OSのファイル操作に依存せず往復できるよう複数手段を用意する）：
  - **JSONテキスト欄**：現在の状態の §3 JSON を常時表示（コピー可）。欄に貼り付けて「テキストを反映」で読み込み。
  - **ファイル書き出し／読み込み**：`.json` のダウンロード／ファイル選択での読み込み。
  - **ブラウザ内保存（localStorage）**：名前を付けて保存し、保存名の一覧から読み込み・削除。キーは `toytank.stage.<name>`。
- **検証表示**：§3制約を満たさない場合は警告（書き出しは可能、ゲーム読込時に弾く）。

### 中4-b 段階1での位置づけ
ToyTank Maker を**先に実装**し、これで作った「標準ステージ1枚」をゲーム本体の動作確認に使う。

## 5. 操作（← 大5）
横持ち・全画面。画面下に2つの操作系を描画する。
- **中5-a 左スティック（移動）**：押した方向へ戦車本体が移動。**デッドゾーン（小5-1）を超えたら方向そのままで最大速度**（少し入れたら全速。アナログの速度可変は廃止）。
  - **戦車らしい旋回**：車体の向き(heading)を入力方向へ `tankTurnRate`（小5-2、約90°/0.5s）で旋回させてから進む。前進と**真逆（バック）はキャタピラなので旋回不要＝即移動**。横方向への新規発進は旋回ぶん遅れる（停止→別方向は約0.5s）。移動中の方向転換は止まらず曲がる。
- **中5-b 右エイムパッド（照準＆射撃）**：
  - ドラッグ中：パッドの向きへ砲塔を向け、**直進の弾道予測線**を表示（§9）。
  - **離した瞬間に1発発射**（その向きへ）。
- **中5-c 即撃ちタップ**：右パッドを引かずに画面を軽くタップ → **進行方向**（停止時は最後に向いた方向）へ1発発射。
- **中5-d 地雷ボタン**：押すと現在位置に地雷を設置（§7）。
- **中5-e 砲塔の向き**：エイム中はパッド方向。非エイム時は進行方向に追従。
- **中5-f 操作モード（スマホ / PC）**：**端末で自動判定**（タッチ端末=スマホ、他=PC）。**手動セレクタは廃止**。
  - スマホ：上記（左スティック＋右エイムパッド＋タップ＋💣）。
  - PC：移動=WASD/矢印、**照準=マウスカーソル**（砲塔と予測線がカーソルへ）、**左クリックでカーソル方向へ発射**、Space=即撃ち（進行方向）、E=地雷。
- **中5-g スプライト**：`public/sprites/` に画像があれば使用（無ければ図形にフォールバック）。タイル(floor/steel/brick/hole)＝64px、戦車は車体`tank_body`＋砲塔`tank_turret`の2枚（白系シルエットをタイプ色で乗算着色、薄アルファ/緑フチ除去、車体=移動方向・砲塔=照準方向で回転）。

**小方針（← 中5、調整可・正は §13／コード）**：小5-1 デッドゾーン（約8px）・小5-2 `tankTurnRate` 旋回速度（約90°/0.5s）・小5-3 `tankSpeed` 最大速度。

## 6. 弾・命中・跳弾・フレンドリーファイア（← 大4・大3）
- **中6-a 弾の基本**：弾は**直進**（速度 `bulletSpeed`（小6-1）既定380px/s、サイズ基準 `bulletRadius`（小6-2）既定9px）。各戦車の**画面内同時発射数に上限**（`maxActiveBullets`（小6-3）既定5）。見た目は進行方向を向いた先端のとがった細長い形状。
- **中6-b 跳弾**：弾は壁／場外境界で反射（軸平行反射＝当たった面に応じて vx か vy を反転）。**反射回数上限** `maxBounces`（小6-4）既定1。上限を超える反射で消滅。
- **中6-c 命中判定の対象**：
  - **壊せるブロック（2）・鋼（1）・場外境界に命中**：いずれも**反射**（上限まで）。**弾では壊せるブロックを破壊しない**（破壊は地雷の爆発のみ）。
  - **穴（3）の上は通過**（反射しない）。射線・爆風も穴を通る（壁＝鋼/ブロックのみが弾・射線・爆風を遮る）。
  - **戦車に命中**：その戦車を**破壊**し、弾は消滅。
- **中6-d フレンドリーファイア**：弾は敵味方・自機を問わず戦車に当たる。
  - 自爆防止の猶予：発射後 `selfGrace`（小6-5）既定150ms は**発射者自身には当たらない**。猶予経過後は、跳ね返った自分の弾でも当たる。
- **中6-e 弾同士の相殺**：**別の射撃グループ**の弾どうしが接触したら（自弾同士・敵弾同士・自↔敵いずれも）**両方消滅＋小爆発**。**同一の一斉射/連射の弾は相殺しない**（各発射＝固有グループ、salvoは1グループ）。判定は両半径の和（`2*bulletRadius`）以内。
- **中6-f 発射直後の停止**：戦車は弾を撃った瞬間から `fireStun`（小6-6）既定0.1s の間、移動できない（全戦車共通）。

**小方針（← 中6、調整可・正は §13／コード）**：小6-1 `bulletSpeed`・小6-2 `bulletRadius`・小6-3 `maxActiveBullets`・小6-4 `maxBounces`・小6-5 `selfGrace`・小6-6 `fireStun`。

## 7. 地雷（← 大5・大4）
- **中7-a 設置**：戦車は地雷を設置（**同時設置数上限** `maxMines`（小7-1）既定2）。半径 `mineRadius`（小7-2）既定18px。
- **中7-b 起動猶予**：`armDelay`（小7-3）既定0.5s 経過後にアクティブ化（中心が赤く点灯）。
- **中7-c 起爆条件**：(a) 信管 `fuse`（小7-4）既定6s 経過、(b) **設置後に弾が当たる**と即起爆、(c) 他の爆発に巻き込まれて**誘爆**。
  - ※「戦車の近接で起爆」は不採用。
- **中7-d 警告表示**：起爆 `mineWarn`（小7-5）既定2s 前から、光（中心）とオーラ（外周リング）が点滅する。
- **中7-e 爆発**：半径 `blastRadius`（小7-6）既定 2.0*cell 内の対象に作用。**戦車を破壊**（FF：敵味方を問わず当たる。**設置者保護は廃止＝起動猶予中でも爆風内なら設置者本人も巻き込まれる**。自分の地雷で弾を受け止めて無傷になる“盾”悪用を防ぐため）、**壊せるブロックを破壊**、**他の地雷を誘爆**、**爆風内にある弾も誘爆して消える**。
  - **壁は越えない**：爆心と対象の間に壁（鋼・壊せる壁）があれば、その対象には効果が及ばない（対象自身が壊せる壁の場合、その壁自体は遮蔽に数えない）。
  - **多重貫通しない**：1回の爆発の影響は**破壊前のタイル**で確定する（壊す壁・巻き込む戦車を先に判定→最後に破壊）。重ねた壊せる壁は手前1枚だけが壊れ、奥は守られる。

**小方針（← 中7、調整可・正は §13／コード）**：小7-1 `maxMines`・小7-2 `mineRadius`・小7-3 `armDelay`・小7-4 `fuse`・小7-5 `mineWarn`・小7-6 `blastRadius`。

## 8. 衝突・移動（← 大1・大4）
- **中8-a 戦車 vs 壁/穴**：戦車を半径 `tankRadius`（小8-1）既定24px の円とし、床以外（鋼1・ブロック2・**穴3**）の矩形と円で判定。めり込まないよう**軸別スライド**。穴も戦車は通れない（落下はしない）。
- **中8-b 戦車 vs 戦車**：自機は敵を半径 tankRadius の円の障害物として扱い貫通を防ぐ。**移動型の敵も「壁＋自機＋他の戦車」を避けて動く**（互いに重ならない・軸別スライド）。
- **中8-c 戦車 vs 場外**：場外セルもソリッド扱いで境界外へ出さない。
- **中8-d 通路幅**：1セル幅の通路（64px）を直径48pxの戦車が通れる。

**小方針（← 中8、調整可・正は §13／コード）**：小8-1 `tankRadius`。

## 10. 敵AI（← 大4）
- **中10-a 標的**：対象は最も近いプレイヤー。
- **中10-b 射線判定（LOS）**：敵→対象の線分が壁セル（1/2）と交差するか。
  - **クリア**：対象へ直接照準して発射（MVPはリード無し）。
  - **遮蔽あり**：**バンクショット**を試みる。
- **中10-c バンクショット（反射1回）**：対象を壁面で鏡像化し、敵→鏡像 が当該壁まで遮られず、かつ反射後 壁→対象 も遮られなければ、鏡像方向へ発射（＝1回反射で対象に当たる射線）。
  - MVPは**場外境界4面での1回反射**を対象とする（内部の鋼壁での反射は将来拡張）。
  - 発射弾の `maxBounces` は1（実際に跳ねる）。
- **中10-d 同士討ち回避**：発射前に弾道（直射・跳弾の反射経路を含む）をシミュレートし、プレイヤーへ届く手前に味方の敵戦車がいると**発射を見送る**（乱戦でも仲間を撃たない）。
- **中10-e 行動パターン**：
  - **stationary**：移動しない。**直射＋バンクショット**。発射間隔 `enemyCooldownStationary`（小10-1）既定2.6s・遅め。
  - **mover**：移動する。**直射のみ（バンクショットなし）**、**照準にばらつき** `enemyAimJitter`（小10-2）既定±0.12rad。発射間隔 `enemyCooldownMover`（小10-3）既定1.8s。
    - **行動軸**を一定間隔（`behaviorMin`〜`behaviorMax`（小10-4））でランダム切替：**戦闘**（接近）／**退避**（離れる・自機が `enemyNear*1.5` 内のときだけ）／**無目的**（徘徊）。重みは概ね 戦闘0.6/徘徊0.3/退避0.1。
    - **壁を擦らず回り込む**：希望方向が塞がれていたら角度を段階的にずらして通れる向きへ進む（軸スライドはしない）。他の戦車・自機とも重ならない。
    - **砲塔は常に自機を向く**（移動方向と分離。壁際でも向きがブレない）。
  - **近接時**：自機が `enemyNear`（小10-5）既定280px 内にいると発射間隔を `nearFireMult`（小10-6）×0.55 に短縮＝攻撃が激しくなる（両タイプ共通）。

### 中10-f 敵タイプ（データ駆動・`stage/enemyTypes.ts` のレジストリ）
- 敵は **タイプ（色）ごとの設定値** を持つ：色／速度(0=静止)／行動性格(guard/approach/kite/balanced/chaser)／発射間隔／発射数／**salvo(同時扇状) or 逐次連射**／反射回数／弾速／バンク可否／**サイズ倍率scale**／照準ばらつき／地雷数／**HP**／透明。
- ステージの `enemy.pattern` はタイプキーを参照。リザルト・区切り画面の「色×数」、撃破集計もタイプ別に自動対応。
- **連射**：`salvo=false`(既定)は同じ発射口から逐次速射（`burstGap` 最小0.13s）。`salvo=true`は複数門の同時扇状発射（ボス）。
- **HP**：被弾/爆風で1減り0で破壊（途中は小ヒット表示）。被弾リスポーンで生存敵のHPは全回復（**ただしボスだけは削ったHPを引き継ぐ**）。
- **サイズ**：`scale` で当たり判定・描画を拡大（ボス=2倍）。発射は**機体半径の外**から（大型機の自滅防止）。
- 実装済タイプ（計13）：stationary/mover（旧）＋ wood/gray/darkgreen/yellowgreen/yellow(地雷4)/pink(3連射)/purple(5連射バンク)/black(超高速追跡2連射ミサイル)/**white(5連射バンク・地雷2・透明)**/**silver(回避特化・銀)**/boss(2倍・**HP12**・5門同時斉射)。**地雷を置くのは yellow(4)/white(2) のみ**（purple/black/silver は地雷なし）。

### 中10-g 移動AIの補助ルール
- **全行動（戦闘/退避/徘徊）でBFS経路探索**：目的セル（戦闘=自機/退避=離れた床/徘徊=ランダム床）への「次の一歩」へ進み、**壁を正確に避ける**（0.3s毎に再計算）。
- **壁越しに突っ込まない**：守備型(balanced/kite)は戦闘中に射線が壁で切れ、かつ自機が `enemyClose`(150px)より遠ければ退避へ切替（攻撃型は退かず回り込む）。
- **ハマり脱出**：同じマスに2秒以上留まったら無目的（目的地を選び直す）へ。
- **砲塔の向き**：射線が通っている時だけ自機へ向け、普段は進行方向を向く。
- **自分の地雷を警戒**：地雷を置くタイプは、自分の地雷の爆風圏へは踏み込まない（離れる方向は可）。※設置者保護は廃止（§7）＝爆風内なら設置者も巻き込まれる。
- 逐次連射は**毎回ランダムな発数**（1〜最大、自機が近いほど最大に寄る）。連射間隔は `burstGap`(0.26s)。発射ごとに `fireStun`(0.1s) 停止。
- **white(白・透明)**：開始 `cloakTime`(1s) は表示、以後は煙とともに本体を描画しない（轍と発射位置で推測）。当たり判定・AIは通常どおり。
- 弾の相殺ルールは §6（別の射撃グループの弾どうし）。
- ※バンクショットの射角計算（鏡像法）は実装の難所。まず場外4面の1回反射に限定して堅実に作る。

**小方針（← 中10、調整可・正は §13／コード）**：小10-1 `enemyCooldownStationary`・小10-2 `enemyAimJitter`・小10-3 `enemyCooldownMover`・小10-4 `behaviorMin`/`behaviorMax`・小10-5 `enemyNear`・小10-6 `nearFireMult`、ほか `burstGap`/`cloakTime`/`enemyClose`/各タイプ設定値（`enemyTypes.ts`）。

## 11. 残機・進行・勝敗（← 大4・大3）
- **中11-a 破壊単位**：全戦車（プレイヤー・敵）とも**1発／爆発で破壊**。
- **中11-b クリア条件**：敵を全滅。
- **中11-c ソロ残機**：`soloLives`（小11-1）既定3。**被破壊で残機-1 → 区切りポーズ（`respawnPause`（小11-2）既定2.2s「ミス！残機×N」）後、自機だけスタート地点に復活**。
  - **倒した敵・壊した壁・進行状況はそのまま維持**（被弾では巻き戻さない）。設置中の地雷・飛行中の弾はクリア。
  - **生き残った敵は初期位置(home)へ戻す**（向き・発射クールダウンもリセット）。倒した敵は復活しない。
  - **残機0でゲームオーバー**。倒した敵などが完全に元に戻るのは「ゲームオーバー／手動リスタート／次ステージ」のときだけ。
  - **ゲームオーバー時リザルト**：色別（敵パターン別）の戦車アイコン×撃破数を表示。撃破数はランを通して集計し、リスタートでリセット。
  - **リザルト選択**：全クリア／残機ゼロのリザルト画面で「**もう一度プレイ／タイトルへ**」をボタンで選べる。
- **中11-d キャンペーン進行**：ステージの敵を全滅でクリア → 次ステージへ（**残機は引き継ぐ**）。最後をクリアで全クリア。リスタート（R/ボタン）は最初のステージから・残機リセット。
  - **残機回復**：**5面ごとに残機+1**（上限 `maxLives`（小11-3）=5）。**+1はその面（5/10/15）のクリア画面に「残機 +1 回復！」を表示**する（次ステージ開始時ではない）。
  - 各ステージ開始時に**区切り画面**（`introPause`（小11-4）既定2.2s）：「ステージN」・出現する敵（色×数）・残機を表示してから開始。
- **中11-e デュオ（段階2）**：片方破壊→**復活なしで待機**（相方生存中は続行）。両方破壊（全滅）で**ステージやり直し**。

**小方針（← 中11、調整可・正は §13／コード）**：小11-1 `soloLives`・小11-2 `respawnPause`・小11-3 `maxLives`・小11-4 `introPause`。

## 12. 段階2（オンライン・デュオ協力）（← 大3・大7、2026-06-27 詳細化）

### 中12-a 同期アーキテクチャ＝ホスト権威＋Cloudflare Durable Objects リレー
- 片方の端末が **ホスト**：既存の TS シミュレーション（§2/§6/§7/§8/§10）を**そのまま実行**し、ゲーム状態の唯一の権威を持つ。
- もう片方は **ゲスト**：自分の入力を送信し、ホストが配る**盤面スナップショット**を受け取って補間描画するだけ（自前ではシミュレーションしない）。
- **リレー＝Cloudflare Durable Objects**：1部屋 = DO 1インスタンス。両者の WebSocket をつなぎ、メッセージを中継するだけ（**sim は持たない**）。WebSocket Hibernation で待機中は課金されない。
- 利点：既存 sim を再利用＝書き直し最小。対CPU協力のためチート耐性は不要 → 権威の偏りは許容。乱数（敵 cd 等）はホストだけが回すため **2台で計算結果を一致させる必要がない（決定論不要）**。
- 旧案（権威サーバー Go で全 sim 実行）は移植コスト大で見送り。EOS はブラウザ JS のリアルタイム P2P 非対応で不可。

### 中12-b 接続・部屋（ロビー）フロー＋役割
- タイトルの **「Co-op マルチ」を解禁**（§1.1-a の Coming Soon を解除）。押すと「**部屋を作る／部屋に入る**」の2択。
- **部屋を作る＝ホスト**：DO に接続 → **合言葉（4文字英数字）** を発行 → 「相手を待っています…」表示。
- **部屋に入る＝ゲスト**：合言葉を入力 → 同じ部屋に接続。**URL 共有も可**（`?room=ABCD` で入室画面に合言葉が入る）。
- **役割は固定**：作成者＝ホスト（権威）、参加者＝ゲスト。途中入替なし。
- **満室**：すでに2人いる部屋への3人目は拒否（「満室です」）。
- 2人そろったら **ホストがステージ開始** → 双方ゲーム画面へ。

### 中12-c やり取りするデータ（プロトコル）
- **ゲスト→ホスト（入力）**：移動軸(x,y)・照準方向・発射イベント・地雷設置イベント。状態は約 **30Hz** で送信、発射/地雷は**イベントとして即時**（取りこぼし防止）。
- **ホスト→ゲスト（盤面スナップショット）**：プレイヤー2台（位置・車体/砲塔向き・生死/待機）・敵配列（位置・向き・タイプ・hp・透明）・弾・地雷・撃破バッテン印・HUD（残機相当/敵数/ステージ番号/状態）。約 **20Hz**。爆発・音は「発生イベント」を送りゲスト側で再現。キャタピラ跡は送らずゲスト側ローカル生成。
- **ロビー系メッセージ**：create-room→code / join-room(code)→ok|full|notfound / peer-joined / peer-left / start(stage)。
- 形式は **JSON・毎回フルスナップショット**（差分・バイナリ化は重ければ後で最適化）。

### 中12-d 補間・遅延（ゲスト側のなめらか化）
- ゲストは **約100ms 過去**を基準に、受信した連続スナップショット間を**線形補間**して 60Hz 描画（位置・向き。角度は最短回り）。
- 敵撃破・弾消滅などの出現/消滅はスナップショット通り即時反映。爆発・音はイベントで再生。
- **自分の戦車も補間表示（先読みなし）**＝MVP。往復遅延ぶんのもたつきが体感で気になれば、将来「自分の戦車だけクライアント予測（先読み＋補正）」を追加する。

### 中12-e スコープ（遊ぶステージ）と生死ルール（← 大4）
- 既存 **20面キャンペーンを協力対応**。2人目の開始位置 **P2 は ToyTank Maker で各面に配置**し `campaignStages.ts` へ焼き込む（フォーマット・エディタは P2 対応済み＝§3/§4）。**P2 未設定のステージは P1 の隣に自動配置**（保険。自作ステージでも協力可）。
- 進め方：**まず1面で疎通確認 → OK で全20面**。**PvP は対象外**（将来）。
- 生死（大4）：1発で破壊。**やられた人は復活なしで待機（観戦）**、相方生存中は続行。**両方やられたら全滅 → そのステージを最初からやり直し**（2人とも復活）。**個別残機は使わない**。FF=ON。敵全滅でクリア。

### 中12-f 切断・終了
- **片方が切断/退出したら、もう片方も終了してタイトルへ**（「相手の接続が切れました」表示）。ホスト/ゲストどちらが抜けても同じ＝**1人継続はしない**。
- ゲーム中の「タイトルへ戻る」も離脱＝相手に通知して双方終了。ロビーで戻ると部屋を閉じる。
- **自動再接続は MVP 対象外**（切れたら終了。将来対応）。部屋は接続ゼロで DO が自動的に片付く。

### 中12-g 開発環境・接続先の切替
- リポジトリ直下に **`relay/`**（`client/` と並列・独立）を新設し、Cloudflare Worker＋Durable Object（TS）を置く。自前 `package.json`・`wrangler.toml`。
- ゲームの接続先は **Vite 環境変数 `VITE_RELAY_URL`** で切替：ローカル=`ws://localhost:8787`（`wrangler dev`）／本番=`wss://…workers.dev`（`wrangler deploy`）。本番 Pages は https のため **wss 必須**。
- デプロイ：クライアント=GitHub Pages 自動／Worker=当面手動 `wrangler deploy`（将来 CI 化）。リレーは中継のみで秘密鍵不要。
- 2人テスト：1台でブラウザ2窓（両方 localhost）→ 片方が部屋作成、もう片方が合言葉で参加。

**小方針（← 中12、調整可・正はコード）**：小12-1 スナップショット送信レート（既定20Hz）・小12-2 入力送信レート（既定30Hz）・小12-3 補間の遅延幅（既定約100ms）・小12-4 合言葉の文字数（既定4）。

## 14. 演出（キャタピラ跡・撃破印・大破演出）（← 大6）
- **中14-a キャタピラ跡**：戦車（自機・移動型の敵）が `trackGap`(9px) 進むごとに、薄い2本のトレッド線を床に残す（最大 `maxTracks`、超過で古いものから消去）。**被弾（残機減）でリセット**＝継続しない。完全リセット（ゲームオーバー/手動リスタート/次ステージ）でも消える。
- **中14-b 撃破バッテン印×**：戦車がやられた座標に×印を残す。**敵は白固定**（視認性のため暗いふちどり）。完全リセットまで残る（被弾リスポーンでは消えない）。
  - 段階2：プレイヤー/味方は**その戦車の色**の×にする。
- **中14-c 大破演出**：戦車が破壊されると、その位置に大きな爆発＋周囲に小爆発（`deathFx` 0.85s）。
  - 自機被弾時は **dying 状態**で演出を見せてから「ミス！区切り→復活」または「ゲームオーバー（リザルト）」へ。演出中は中央表示なし・敵/弾は一時停止。
  - 敵撃破時はプレイ継続のまま同じ爆発を出す。

## 15. チュートリアル（← 大10）
初見プレイヤーが本編前に基本操作を体験して理解できる練習モード。**実際に動かして覚える（達成検知ステップ制）＋ 要所での説明オーバーレイ** の併用。

- **中15-a 起動・導線**：タイトルの **「ソロ」直下に小さく「チュートリアル」** 項目（§1.1-a）。選択でチュートリアルを開始する。
  - 画面状態は `game` を再利用し、`Game` に**チュートリアルモード**フラグを持たせる（通常プレイと共通の描画/操作系を使う）。
- **中15-b 進行＝達成検知ステップ制**：各ステップで操作を1つ教え、**その操作の達成イベントを検知したら次のステップへ**進む。ステップ列：
  1. **移動**：一定距離動いたら達成。
  2. **即撃ち**（軽いタップ/クリック＝進行方向へ発射）：**1発撃つ**と達成。
  3. **エイム発射**（右エイムパッド／PCマウス照準→離して発射）：**狙って1発撃つ**と達成。
  4. **地雷**（💣ボタン）：設置→近くの**壊せる壁を爆破**すると達成（壊せる壁はここで体得）。
  5. **仕上げ**：木戦車（`wood`）1体を撃破すると**クリア**。
  - **壊せる壁の破壊は④（地雷）で教える**（弾では壊せる壁を壊せない＝地雷の爆発でのみ破壊する仕様。§6）。**穴・跳弾は扱わない**。
- **中15-c 安全設計**：チュートリアル中は**プレイヤー無敵**（被弾しても残機が減らない・dying 演出に入らない）。仕上げの敵は **木戦車（ほぼ撃ってこない）1体のみ**にし、ストレスなく完走できるようにする。
- **中15-d 説明オーバーレイ**：各ステップで短いテキスト＋**操作表記を端末で出し分け**（PC＝クリック/WASD/マウス、スマホ＝タップ/左パッド/右パッド/💣）。§5・§1.1-c に準拠。
- **中15-e スキップ・終了**：画面の隅（または⚙ギア内）に**スキップ**を用意し、押すとタイトルへ戻る。全ステップ完了（敵撃破）でも「チュートリアル完了！」表示後に**タイトルへ戻す**（本編は自分で「ソロ」を選ぶ）。
- **中15-f データ**：チュートリアル専用ステージは**コードの専用定義（`tutorialStage.ts` 想定）で保持し読み取り専用**。公式キャンペーン（`campaignStages.ts`）／自作（localStorage）とは混ぜない（§1.1-b の分離方針）。

**小方針（← 中15、調整可・正はコード）**：小15-1 移動達成の距離しきい値・小15-2 各ステップのオーバーレイ文言・小15-3 練習ステージのレイアウト（的/壊せる壁/敵の配置）。
