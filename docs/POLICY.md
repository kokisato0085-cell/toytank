# ToyTank 方針書（前提条件・大方針）

ToyTank の設計書は **大方針 → 中方針 → 小方針** の3階層で構成する。

- **大方針（大N）**＝本書。ゲームの方向性・前提（why/what）。滅多に変えない。
- **中方針（中N）**＝[BasicDesign.md](BasicDesign.md)。各システムの設計判断（how）。各 大N から `→ 中N` で辿る。
- **小方針（小N-n）**＝[BasicDesign.md](BasicDesign.md) 各節末尾＋§13マスター表。調整可能な数値・閾値（値の正はコード）。

方針は「前提条件（P）」の上に成り立ち、前提が崩れたら方針も再評価する。

## 前提条件

| 番号 | 内容 | 確度 |
|---|---|---|
| P1 | 個人開発（一人で設計・実装する） | 確定 |
| P2 | スマホ向け Web/PWA（インストール審査なし・URLで配布／「ホーム画面に追加」でアプリ風） | 確定 |
| P3 | 「戦車！」のオマージュだが、名称・画像・音はすべて自作（既存IP素材は不使用） | 確定 |
| P4 | 開発はウォーターフォール（設計先行）。DEV_RULES R1〜R8 を遵守 | 確定 |

## 大方針

### 大1 視点・カメラ — 確定 → 中1・中2・中8
**2D見下ろし（トップダウン）**。ブロスタのような見下ろしシューターの見た目・操作感。3Dは使わない。

### 大2 技術スタック — 確定（2026-06-27 改定）
**Web/PWA × 2D Canvas（TypeScript）**。クライアントは Vite。
オンライン協力の同期は**ホスト権威＋Cloudflare Durable Objects のリレー（TypeScript）**で実現する（段階2で追加）。ソロはクライアント単体で完結。
※ 当初は「権威サーバーは Go」としていたが、対CPU協力ではサーバー権威の主目的（チート防止）が薄く、TS sim を Go へ全面移植するコストが大きいため見直した（[[大3]]）。Cloudflare は無料枠で実質¥0・コールドスタート無し・circlefee で利用実績あり・ゲームも TS でスタック統一。

### 大3 マルチプレイ形態 — 確定 → 中6・中11・中12（2026-06-27 同期方式を改定）
- **段階目標（MVP）**：1ステージを「ソロ（オフライン1人）」または「オンライン・デュオ（2人協力）」の好きな方で攻略。**フレンドリーファイアON**。
- **最終目標**：上記に **PvP対戦** を追加。
- 協力はPvPより同期が容易なため、「協力で先にオンライン化 → 後でPvP」の順とする。
- **同期方式＝ホスト権威＋Cloudflare Durable Objects リレー（改定）**：片方の端末が「ホスト」で既存 TS sim を実行し権威を持つ。もう片方「ゲスト」は入力を送り、ホストが配る盤面スナップショットを補間描画する。Cloudflare Durable Objects は「部屋＝DO 1個」として WebSocket を中継するだけ（sim は持たない）。詳細は [BasicDesign.md](BasicDesign.md) §12。
  - 旧案（権威サーバー Go で全 sim を実行）は見送り。EOS も不可（ネイティブ C SDK 中心でブラウザ JS のリアルタイム P2P 非対応）。

### 大4 ゲーム構成・目的 — 確定 → 中1・中6・中7・中8・中10・中11
- **クリア条件**：ステージのCPU戦車を全滅。
- **残機（原作準拠）**：1発で破壊。ソロ＝残機3でやられたらステージ最初からやり直し。デュオ＝片方やられたら復活なしで待機（相方生存中は続行）、両方やられた（全滅）で最初からやり直し。
- **敵の行動パターン（MVP）**：①止まって撃つ ②移動して撃つ。加えて、遮蔽物でプレイヤーへの直射が通らないとき、**壁の反射を計算して跳弾でプレイヤーを狙うバンクショット**を行う。将来は原作に近い多彩なパターンへ拡張。

### 大5 操作系 — 確定 → 中5・中7・中9
ブロスタ（スナイパー型）に近いハイブリッド操作。
- **左スティック＝移動**（動き撃ち可）。
- **右エイムパッド＝敵方向へ引っ張り、離した瞬間に発射**。エイム中は**直進の弾道予測線を表示**（反射は予測しない）。
- **軽いタップ＝照準なしの即撃ち → 進行方向へ発射**。
- **ためて強い一撃は採用しない**。
- **地雷ボタン**を備える。

### 大6 アセット・ビジュアル方針 — 暫定 → 中14
自作アセット・ブロスタ風の見下ろし。**まず仮グラフィック（丸・四角）で挙動を作り、後から絵を差し替える**。色味・テイストは追って確定。

### 大7 スコープ・段階計画 — 確定 → 中4・中12
- **段階1（ソロ・オフラインMVPコア）**：移動・エイム・即撃ち・跳弾・地雷・壊せる/壊せない壁・遮蔽物時のみのプレイヤー予測線・敵2パターン＋バンクショット（反射1回）・残機3・全滅でクリア。**ToyTank Maker（エディタ）を先行実装**し、標準ステージを1枚作れる状態にする。
- **段階2（オンライン・デュオ協力）**：既存20面を2人協力。**ホスト権威＋Cloudflare Durable Objects リレー**でリアルタイム同期、合言葉（コード/URL）で部屋参加、FF=ON。やられたら待機・両方全滅でステージ最初からやり直し。まず1面で疎通確認 → 全20面へ。詳細は [BasicDesign.md](BasicDesign.md) §12。
- **最終（PvP＋拡充）**：PvP対戦・敵パターン拡充・ステージ追加・マッチング等。

### 大8 ステージのデータ駆動 ＋ エディタ — 確定 → 中2・中3・中4
ステージ（マップ・敵配置）は**データファイル（JSON）**で持ち、専用エディタ **ToyTank Maker** でマウス作成する。ゲーム本体はそのJSONを読んでステージを生成する。詳細は [BasicDesign.md](BasicDesign.md) の「ステージデータ形式」「ToyTank Maker」を参照。

### 大9 製品化・公開（提出向け）— 確定 → 中1.1
段階1を「ソロMVPコア」だけでなく **公開URLで誰でも即遊べる製品** まで広げる。
- **GitHub Pages へ自動デプロイ**（main push 毎にビルド→公開）。実機リンクで配布。
- **タイトル（モード選択）シェル**：ソロ／自作ステージで遊ぶ／ToyTank Maker／設定／Co-op・PvP（Coming Soon）。
- **設定画面**：全体音量・ミュート・フルスクリーン切替・操作ガイド（将来：操作カスタム）。
- **タイトル背景に戦車AI同士のデモ（アトラクトモード）**。
- **ダークテーマで統一したUI**。
- **セキュリティCI**（semgrep / gitleaks）を push 毎に実行（検出で停止）。
- 端末別の画面：スマホ＝横画面・没入フルスクリーン＋高精細(DPR)、PC＝拡大・中央寄せ。詳細は [BasicDesign.md](BasicDesign.md) §1.1・§5。

### 大10 チュートリアル — 確定 → 中1.1・中15
初見プレイヤー（ポートフォリオ閲覧者を含む）が本編前に**基本操作を体験して理解できる**チュートリアルを用意する。
- **形式＝併用**：実際に動かして覚える**練習ステージ**（達成検知ステップ制）＋ 要所での短い**説明オーバーレイ**。
- **対象＝PC・スマホ両対応**（操作表記を端末で出し分け。§5・中1.1-c に準拠）。
- **プレイヤー無敵**でストレスなく完走できる安全設計。
- **教える範囲**：移動 → 即撃ち → エイム発射 → 地雷 → （仕上げ）敵撃破。**壊せる壁は即撃ち/地雷の中で体得**させる。**穴・跳弾は扱わない**。
- **導線**：タイトルの「ソロ」直下に「チュートリアル」項目。**途中スキップ可**。**クリア後はタイトルへ戻す**。
- **データ分離**：チュートリアル専用ステージはコードの専用定義で保持し、公式キャンペーン／自作とは混ぜない（中1.1-b の分離方針）。詳細は [BasicDesign.md](BasicDesign.md) §15。
