# ToyTank 方針書（前提条件・大方針）

本書は ToyTank の上位方針を定める。下位の具体仕様は [BasicDesign.md](BasicDesign.md) に展開する。
方針は「前提条件（P）」の上に成り立ち、前提が崩れたら方針も再評価する。

## 前提条件

| 番号 | 内容 | 確度 |
|---|---|---|
| P1 | 個人開発（一人で設計・実装する） | 確定 |
| P2 | スマホ向け Web/PWA（インストール審査なし・URLで配布／「ホーム画面に追加」でアプリ風） | 確定 |
| P3 | 「戦車！」のオマージュだが、名称・画像・音はすべて自作（既存IP素材は不使用） | 確定 |
| P4 | 開発はウォーターフォール（設計先行）。DEV_RULES R1〜R8 を遵守 | 確定 |

## 大方針

### 大1 視点・カメラ — 確定
**2D見下ろし（トップダウン）**。ブロスタのような見下ろしシューターの見た目・操作感。3Dは使わない。

### 大2 技術スタック — 確定
**Web/PWA × 2D Canvas（TypeScript）**。クライアントは Vite。
オンライン協力／PvP のための**権威サーバーは Go**（段階2以降で追加）。ソロはクライアント単体で完結。

### 大3 マルチプレイ形態 — 確定
- **段階目標（MVP）**：1ステージを「ソロ（オフライン1人）」または「オンライン・デュオ（2人協力）」の好きな方で攻略。**フレンドリーファイアON**。
- **最終目標**：上記に **PvP対戦** を追加。
- 協力はPvPより同期が容易なため、「協力で先にオンライン化 → 後でPvP」の順とする。

### 大4 ゲーム構成・目的 — 確定
- **クリア条件**：ステージのCPU戦車を全滅。
- **残機（原作準拠）**：1発で破壊。ソロ＝残機3でやられたらステージ最初からやり直し。デュオ＝片方やられたら復活なしで待機（相方生存中は続行）、両方やられた（全滅）で最初からやり直し。
- **敵の行動パターン（MVP）**：①止まって撃つ ②移動して撃つ。加えて、遮蔽物でプレイヤーへの直射が通らないとき、**壁の反射を計算して跳弾でプレイヤーを狙うバンクショット**を行う。将来は原作に近い多彩なパターンへ拡張。

### 大5 操作系 — 確定
ブロスタ（スナイパー型）に近いハイブリッド操作。
- **左スティック＝移動**（動き撃ち可）。
- **右エイムパッド＝敵方向へ引っ張り、離した瞬間に発射**。エイム中は**直進の弾道予測線を表示**（反射は予測しない）。
- **軽いタップ＝照準なしの即撃ち → 進行方向へ発射**。
- **ためて強い一撃は採用しない**。
- **地雷ボタン**を備える。

### 大6 アセット・ビジュアル方針 — 暫定
自作アセット・ブロスタ風の見下ろし。**まず仮グラフィック（丸・四角）で挙動を作り、後から絵を差し替える**。色味・テイストは追って確定。

### 大7 スコープ・段階計画 — 確定
- **段階1（ソロ・オフラインMVPコア）**：移動・エイム・即撃ち・跳弾・地雷・壊せる/壊せない壁・遮蔽物時のみのプレイヤー予測線・敵2パターン＋バンクショット（反射1回）・残機3・全滅でクリア。**ToyTank Maker（エディタ）を先行実装**し、標準ステージを1枚作れる状態にする。
- **段階2（オンライン・デュオ協力）**：同ステージを2人協力。権威サーバー（Go）＋リアルタイム同期、コード/URLで部屋参加、FF=ON。
- **最終（PvP＋拡充）**：PvP対戦・敵パターン拡充・ステージ追加・マッチング等。

### 大8 ステージのデータ駆動 ＋ エディタ — 確定
ステージ（マップ・敵配置）は**データファイル（JSON）**で持ち、専用エディタ **ToyTank Maker** でマウス作成する。ゲーム本体はそのJSONを読んでステージを生成する。詳細は [BasicDesign.md](BasicDesign.md) の「ステージデータ形式」「ToyTank Maker」を参照。

### 大9 製品化・公開（提出向け）— 確定
段階1を「ソロMVPコア」だけでなく **公開URLで誰でも即遊べる製品** まで広げる。
- **GitHub Pages へ自動デプロイ**（main push 毎にビルド→公開）。実機リンクで配布。
- **タイトル（モード選択）シェル**：ソロ／自作ステージで遊ぶ／ToyTank Maker／設定／Co-op・PvP（Coming Soon）。
- **設定画面**：全体音量・ミュート・フルスクリーン切替・操作ガイド（将来：操作カスタム）。
- **タイトル背景に戦車AI同士のデモ（アトラクトモード）**。
- **ダークテーマで統一したUI**。
- **セキュリティCI**（semgrep / gitleaks）を push 毎に実行（検出で停止）。
- 端末別の画面：スマホ＝横画面・没入フルスクリーン＋高精細(DPR)、PC＝拡大・中央寄せ。詳細は [BasicDesign.md](BasicDesign.md) §1.1・§5。
